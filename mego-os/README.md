# Mego OS — نظام إدارة محل ميجو

نظام إدارة شامل لمحل **ميجو** لخدمات الصيانة والطباعة والدعم التقني. شعار المحل: **"خدمتك في مكان واحد"**.

تطبيق ويب عربي (RTL) كامل لإدارة العملاء، طلبات الصيانة، أعمال الطباعة، المخزون، المبيعات، المصاريف، اشتراكات الدعم الشهري للمحلات، المتابعات، والتقارير المالية — مع طباعة إيصالات وفواتير، وروابط واتساب جاهزة بالرسائل العربية.

## المميزات المكتملة

- تسجيل دخول وحماية صفحات عبر Supabase Auth، بأدوار Admin/Staff.
- **العملاء**: بحث فوري، سجل كامل للعميل (طلبات، فواتير، اشتراكات).
- **طلبات الصيانة**: حالة الطلب، الأولوية، الأسعار التقديرية والنهائية، الدفعات، إيصال طباعة قانوني.
- **الطباعة والبانرات**: طلبات طباعة/تصوير/بانرات مع إيصال طباعة.
- **الخدمات**: قائمة أسعار قابلة للتعديل المباشر.
- **المخزون**: كميات، حد أدنى للتنبيه، خصم تلقائي عند البيع.
- **المبيعات**: فاتورة بيع (خدمات/منتجات مختلطة) مع خصم المخزون وفاتورة طباعة.
- **المصاريف**: تسجيل وتصنيف مع إجمالي الشهر.
- **اشتراكات المحلات (Business Care)**: باقات شهرية، عدد الزيارات، تجديد، رسائل تذكير واتساب.
- **المتابعات**: جدولة متابعات للعملاء مع تذكير واتساب.
- **التقارير**: دخل/مصاريف اليوم والشهر، أكثر الخدمات طلبًا، طلبات مفتوحة، أرصدة متبقية.
- **الإعدادات**: بيانات المحل المستخدمة في الإيصالات (للمدير فقط).
- **الرئيسية (Dashboard)**: نظرة عامة فورية + إجراءات سريعة.
- رسائل واتساب جاهزة بالنص العربي المطلوب لكل سيناريو (استلام، موافقة، جاهز، تسليم، تجديد اشتراك، تذكير متابعة).

## التقنيات

- **Next.js 16** (App Router, TypeScript, Turbopack)
- **Tailwind CSS v4** بخط Cairo وتصميم RTL كامل
- **Supabase** (Postgres + Auth + RLS)

## الإعداد المحلي

```bash
cd mego-os
npm install
cp .env.example .env.local   # ضع بيانات مشروع Supabase
npm run dev
```

التطبيق يعمل على `http://localhost:3000`.

## إعداد Supabase

1. أنشئ مشروع جديد على [supabase.com](https://supabase.com).
2. من **SQL Editor** نفّذ الملفين بالترتيب:
   - `supabase/schema.sql` — الجداول، الأنواع، الفهارس، الـ triggers، وسياسات RLS.
   - `supabase/seed.sql` — بيانات أولية (إعدادات المحل والخدمات الافتراضية).
3. من **Project Settings → API** خذ `Project URL` و `anon public key` وضعهم في `.env.local`.
4. أول مستخدم يسجل (Sign Up) يحصل تلقائيًا على دور `staff`. لترقيته لـ `admin`، نفّذ في SQL Editor:
   ```sql
   update profiles set role = 'admin' where id = '<user-id>';
   ```

## النشر على Vercel

1. اربط المستودع بـ Vercel، واجعل **Root Directory** = `mego-os`.
2. أضف متغيرات البيئة `NEXT_PUBLIC_SUPABASE_URL` و `NEXT_PUBLIC_SUPABASE_ANON_KEY` في إعدادات المشروع على Vercel.
3. Deploy. لا حاجة لإعدادات إضافية — Next.js App Router يعمل تلقائيًا على Vercel.

## المتغيرات السرية المطلوبة

| المتغير | الوصف |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | رابط مشروع Supabase |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | المفتاح العام (anon) |
| `SUPABASE_SERVICE_ROLE_KEY` | اختياري — للسكربتات الإدارية فقط، **لا يجب كشفه أبدًا للمتصفح** |

## تحسينات مقترحة للمستقبل

- إرسال رسائل واتساب تلقائيًا عبر WhatsApp Business API بدل روابط `wa.me` اليدوية.
- تنبيهات تلقائية (إشعارات/بريد) عند اقتراب تجديد الاشتراكات أو نقص المخزون.
- صلاحيات أكثر تفصيلاً (مثلاً صلاحية تعديل الأسعار فقط للمدير).
- رسم بياني (Charts) للدخل والمصاريف على مدار الزمن في صفحة التقارير.
- دعم رفع صور للأجهزة المستلمة وإرفاقها بطلب الصيانة.
- تصدير التقارير إلى Excel/PDF.
